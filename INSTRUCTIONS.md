# INSTRUCTIONS.md – agents‑website

## Описание проекта
`agents‑website` — статический дашборд, отображающий статус агентов OpenClaw, их последние задачи и состояние ComfyUI. Данные получают из JSON‑файлов в каталоге `data/`, которые регулярно обновляются сервисом `real_time_updater.py`.

## Структура проекта
```
agents‑website/
├─ assets/                       # дополнительные файлы (иконки, изображения)
├─ css/
│   └─ styles.css                # стили сайта
├─ js/
│   └─ script.js                 # клиентский скрипт, загружает статус и задачи
├─ data/
│   ├─ status.json                # текущий статус агентов + ComfyUI
│   └─ agents_tasks_detail_human.json  # человекочитаемый список последних 5 задач на каждый агент
├─ index.html                     # главная страница (авто‑обновление каждые 30 сек), использует новый layout с `tasks-bar`
├─ index_backup.html              # резервная копия старого шаблона
├─ scripts/
│   └─ check_comfy_status.py      # однократная проверка доступности ComfyUI (записывает в data/status.json)
├─ service/
│   └─ real_time_updater.py       # Python‑сервис, каждые 2 мин обновляет статус, задачи и ComfyUI (ограничение памяти 200 MiB)
├─ manage_server.sh               # интерактивный менеджер systemd‑служб (веб‑сервер и обновлятор)
├─ setup_agents_website.sh        # скрипт создания/перезапуска systemd‑юнита `agents-website.service`
└─ INSTRUCTIONS.md                # справка проекта (этот файл)
```

## Обновления UI

Ниже перечислены изменения, внесённые в UI‑слой проекта:

- **`css/styles.css`** теперь содержит блок стилей `/* ── Блок задач ── */` с классами `.tasks-bar`, `.tasks-bar .status-item` и `.tasks-bar .tasks-section`. Эти стили выравнивают блоки задач в одной строке и делают их гибко растягиваемыми.
- **`index.html`** использует новый контейнер `<div class="status-bar tasks-bar">` для отображения задач трёх агентов (Маришка, OpenClaw, ComfyUI). Каждый блок имеет заголовок `Задачи <агент>` и отдельный контейнер `.tasks-section` для списка задач.
- Клиентский скрипт `js/script.js` автоматически наполняет эти секции, а стили обеспечивают визуальное разделение и адаптивность.

## Назначение ключевых компонентов
- **`setup_agents_website.sh`** – создает (или перезапускает) systemd‑юнит `agents-website.service`, который обслуживает каталог `index.html` через `python3 -m http.server` на порту 8000.
- **`manage_server.sh`** – интерактивный меню‑менеджер для удобного управления двумя сервисами: веб‑сервером `agents-website.service` и обновляющим сервисом `agents-website-updater.service` (остановка, запуск, просмотр логов).
- **`service/real_time_updater.py`** – Python‑демон, каждые 120 сек собирает статус OpenClaw, проверяет доступность ComfyUI (`http://192.168.0.113:8188`) и атомарно записывает результаты в `data/status.json` и `data/agents_tasks_detail_human.json`. Ограничение памяти ≈ 200 MiB.
- **`scripts/check_comfy_status.py`** – однократный скрипт для быстрой проверки состояния ComfyUI (если нужен быстрый чек без постоянного сервиса).
- **`css/styles.css`** и **`js/script.js`** – стили и клиентский код, отвечающие за отображение статуса, цветовую разметку, новый layout задач и автоматическое обновление страницы каждые 30 сек.
- **`data/status.json`** – JSON с текущим статусом агентов (online/offline/inWork/… и статус ComfyUI).
- **`data/agents_tasks_detail_human.json`** – человекочитаемый список последних 5 задач для каждого агента (метки, статус, время начала, длительность).
- **`index.html`** – основной UI‑шаблон, использующий `css/` и `js/`.
- **`index_backup.html`** – резервная версия старого шаблона, сохраняется на случай отката.

## Как использовать
1. **Быстрая локальная проверка** (без systemd)
   ```bash
   cd /home/dmitriy/.openclaw/workspace/Projects/agents-website
   python3 -m http.server 8000   # запустит простой HTTP‑сервер
   ```
   Затем откройте в браузере `http://localhost:8000/`.

2. **Полная инициализация через systemd**
   ```bash
   cd /home/dmitriy/.openclaw/workspace/Projects/agents-website
   ./setup_agents_website.sh   # создаст и запустит службу agents-website.service
   ```
   После этого сервис будет обслуживать `index.html` на порту 8000 и автоматически перезапустится после перезагрузки системы.

3. **Запуск обновляющего сервиса**
   ```bash
   # Автоматически через systemd (рекомендуется)
   sudo systemctl enable --now agents-website-updater.service
   # Либо вручную в фоне
   python3 service/real_time_updater.py &
   ```
   Сервис каждые 2 минуты собирает статус OpenClaw и проверяет ComfyUI.

4. **Однократная проверка ComfyUI**
   ```bash
   ./scripts/check_comfy_status.py
   ```
   Скрипт запишет актуальное состояние в `data/status.json`.

5. **Управление сервисами через меню**
   ```bash
   ./manage_server.sh
   ```
   Позволяет останавливать/запускать веб‑сервер и обновляющий сервис, а также просматривать их логи.

6. **Открытие дашборда**
   После запуска веб‑сервера откройте в браузере `http://<host>:8000` (по умолчанию порт 8000). Страница будет автоматически обновляться каждые 30 сек., показывая актуальный статус и задачи.
